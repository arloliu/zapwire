# zapwire v1 — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development
> (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps
> use checkbox (`- [ ]`) syntax for tracking. Execute the phase files **in order**:
> `00` → `01` → `02` → `03` → `04`.

**Goal:** Build a high-performance zap `WriteSyncer` that ships structured logs to log
processors (Fluentd / Fluent-bit / Vector via Fluent Forward msgpack, and
Vector / Logstash / OTel Collector via NDJSON) over UDS or TCP, with configurable
sync/async delivery and never-block drop-on-stall semantics.

**Architecture:** A stdlib-based core (`zapwire`) owns the connection manager (background
reconnect with backoff), the bounded-write dispatch (sync = write-per-log; async = ring
buffer + batched flush), and two small interfaces — `Encoder` (zap bytes → per-entry wire
payload) and `Framer` (`[][]byte` payloads → one wire frame). Per-processor subpackages
implement those interfaces and provide presets: `fluent/` (msgpack Forward, owns
`tinylib/msgp`) and `ndjson/` (newline-delimited, stdlib). The design comes from
`docs/design/2026-06-07-zapwire-design.md`; read it first.

**Tech Stack:** Go 1.25, `go.uber.org/zap/zapcore` (the integration target),
`github.com/tinylib/msgp` (msgpack codegen, quarantined in `fluent/`),
`github.com/stretchr/testify` (tests), isolated `golangci-lint` v2 via a tool module.

---

## Dependency policy (refinement of the spec)

The spec says "stdlib-only core." Refined for implementation: **`zapcore` is the framework
baseline**, not a third-party processor dependency — every zap consumer already has it.

| Package | Allowed imports |
|---|---|
| `zapwire` (root) | stdlib + `go.uber.org/zap/zapcore` |
| `ndjson/` | stdlib + `go.uber.org/zap/zapcore` |
| `fluent/` | stdlib + `zapcore` + `github.com/tinylib/msgp` |
| (future) `otlp/` | own `go.mod` (grpc + protobuf) — see design §11 |

The rule that matters: **no log-processor-format dependency (msgp, grpc, protobuf) leaks
into root or `ndjson`.** An NDJSON-only consumer never compiles `tinylib/msgp`.

## Execution handoff

After `00`–`04` are complete, the writer is usable as:

```go
ws, _ := fluent.NewWriter(zapwire.UDS("/var/run/fluent.sock"), fluent.WithTag("app.logs"))
core := zapcore.NewCore(zapcore.NewJSONEncoder(cfg), ws, zap.InfoLevel)
logger := zap.New(core)
```
